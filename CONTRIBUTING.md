# Contributing Guidelines

Thank you for considering contributing to Dynamic Source Loader!

## How to Contribute

### 1. Report Bugs
- Use the GitHub Issues tracker
- Include system information, Python version, and error logs
- Provide steps to reproduce

### 2. Suggest Features
- Check existing issues first
- Explain the use case and benefits
- Consider implementation complexity

### 3. Submit Code
1. Fork the repository
2. Create a feature branch
3. Write your code with tests
4. Ensure all tests pass
5. Submit a pull request

## Development Standards

### Code Style
- Use Black for formatting
- Maximum line length: 88 characters
- Use type hints for all function signatures
- Document public APIs with docstrings

### Testing
- Write unit tests for new features
- Maintain or improve test coverage
- Test on multiple Python versions (3.8+)

### Documentation
- Update README.md for user-facing changes
- Add docstrings for new functions/classes
- Update type hints

## Pull Request Process
1. Update the CHANGELOG.md
2. Update documentation if needed
3. Add tests for new functionality
4. Ensure CI passes
5. Get at least one review

## Code of Conduct
Be respectful and inclusive. We follow the Python Community Code of Conduct.
